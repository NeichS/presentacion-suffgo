    Introducción

    Suffgo es una plataforma para toma de decisiones en grupos, inspirada en la idea de “suffrage on the go”. Permite crear salas, propuestas y opciones de votación con reglas formales configurables, orientada a equipos y organizaciones.

    • Objetivo: facilitar/agilizar procesos deliberativos transparentes y ágiles.
    • Usuarios: equipos de trabajo, instituciones, comunidades.
    • Features alcanzadas: autenticación, gestión de salas, propuestas y votos; votación tiempo real y reportes post votación.

    ¿Qué hace Suffgo?

    • Alta y administración de usuarios.
    • Creación y gestión de salas de votación, customizacion de la misma.
    • Creación de propuestas con múltiples opciones de voto.
    • Configuraciones formales: tiempos, quórum, reglas y permisos.
    • Tablero de resultados y exportación de reportes.

    ¿Qué no hace Suffgo?

    • No garantiza anonimato del voto.
    • No reemplaza un padrón oficial.
    • No válida identidad con el RENAPER.

    Proceso de desarrollo

    Metodologías

    • Iteraciones cortas (cadencia semanal) con Kanban
    • Scrum a medias: reuniones semanales.

    Git & ramas

    • Estrategia main + develop + forks.
    • Code review a la hora mergear a develop

    Tablero Kanban (Huly)

    • Separamos tareas por backend, frontend y documentación
    • Columnas: ToDo → In Progress → PR Pending→ Testing → Done.
